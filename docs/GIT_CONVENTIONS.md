# Git 運用方針（GIT_CONVENTIONS）

> 自律実行モードで作成したため、推奨デフォルトを採用。異論があれば更新して以降を追従させる。

## Git 管理

- 本プロジェクトは Git で管理する（現在未初期化。実装開始前に `git init` する）。
- デフォルトブランチ: `main`。`main` は常に起動可能・テストグリーンを保つ。

## ブランチ運用

- 1タスク = 1機能 = 1ブランチ = 1PR（TASKS.md のタスク粒度に従う）。
- ブランチ名: `feature/<タスクID>-<slug>`（例: `feature/T03-sake-detail`）。
- バグ修正: `fix/<slug>`、ドキュメントのみ: `docs/<slug>`。

## コミット規約

- Conventional Commits に従う: `feat:` `fix:` `docs:` `test:` `refactor:` `chore:` `ci:`。
- 件名は日本語可。例: `feat: 日本酒詳細ページを追加`。
- 粒度: 「動く状態を保つ最小の意味単位」。WIP コミットを main 系列に残さない。

## 禁止事項

- **シークレット・`.env*`・API キーのコミット禁止**（`.gitignore` で最初に防ぐ）。
- `main` への直接 push（実装フェーズ以降は PR 経由）。
- `--force` push（個人開発でも履歴書き換えは feature ブランチ内に限る）。

## PR 運用

- PR の提出・マージは、ユーザー承認（2026-07-04）により AI が自動で行う。
- マージ前に CI がグリーンであることを必ず確認してからマージする。
- `main` へ直接 push できないため、**直前タスクの状態更新（TASKS.md の「完了」化）や
  運用規約の追記は、次タスクの PR の冒頭コミットに含めてよい**（構造上マージ後にしか
  書けないため。T02 レビュー指摘 S-1 を受けて明文化）。

## ドキュメント運用

- コードの変更が設計ドキュメント（docs/）と食い違う場合、同じ PR でドキュメントも更新する。
